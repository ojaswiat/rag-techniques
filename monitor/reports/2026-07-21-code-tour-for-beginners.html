3The 4 Files That Matter

Everything else is scaffolding. These four Python files, inside project/, are Phase 1's actual work.

project/config.py
Job: the settings sheet. It says which AI model handles which job, how many things can run at once, and a safety flag for "test mode."
  • MODEL_ROUTING — a lookup table: "generator" uses one AI model, "judge" uses another, etc. Kept separate on purpose so no AI ever grades its own homework.
  • LOCAL_TEST_THROTTLE — a true/false switch. When true, everything only processes 3 items at a time (for safe testing) instead of thousands.
  • GROQ_MAX_CONCURRENCY = 5 — never more than 5 things happening with the AI at once, so we don't get rate-limited.
project/database_manager.py
Job: the filing cabinet. Everything the project remembers — documents, questions, answers, scores — is stored in one file (a small database) that this module reads and writes.
  • init_db() — builds the 5 empty drawers (tables) in the filing cabinet, if they don't exist yet.
  • insert_node() — files away one chunk of a document.
  • get_nodes_by_document() — fetches every chunk that belongs to one document.
  • upsert_result() — records one experiment's result (which AI pipeline answered a question, how well it did).
  • get_completed_keys() — checks "what have we already done?" so if the program crashes, it picks up where it left off instead of redoing work.
  • _dumps() / _loads() — small helpers that convert lists into text (and back), because the filing cabinet can only store plain text.
project/groq_client.py
Job: the phone line to the AI. Every single call to the AI provider (Groq) goes through this one function, so retries and rate-limit rules apply everywhere consistently.
  • call_groq() — "ask the AI this, using this model." If the AI says "slow down" (a 429 error), it automatically waits and tries again instead of crashing.
  • _is_rate_limit_error() — a helper that recognizes the specific "slow down" error so it knows when to retry.
project/loop_template.py
Job: the training-wheels switch. Every future script that processes lots of items copies this exact pattern, so nobody accidentally runs a giant, expensive job by mistake while still debugging.
  • apply_throttle() — if test mode is on, only keep the first 3 items from a list.
  • throttle_limit_clause() — builds the text "LIMIT 3" for use in database queries.

Notice database_manager.py, groq_client.py, and loop_template.py don't talk to each other — they each only depend on config.py. That's on purpose: each file has exactly one job, so you can understand (or fix) one without needing to understand the others.

6Where to Start Reading (in order)

  2. project/config.py — shortest file, sets up everything else. Read this first.
  3. project/database_manager.py — the filing cabinet, a bit longer but each function is short and separate.
